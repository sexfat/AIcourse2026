Prompt 範例

// copy & paste into Claude
Prompt 1 — 生成 HTML + Tailwind 佈局
角色：你是一個資深的前端工程師與 UI 設計師。 任務：請使用單一 HTML 檔案（內嵌 Tailwind CSS CDN 與原生 JavaScript）建立一個「美食搜尋與卡路里紀錄」 的 Dashboard 介面。 版面佈局要求（RWD 行動優先）： 1. 頂部大盤：呈現今日卡路里儀表板，包含 「目標 2000 kcal」、「已攝取 0 kcal」、「剩餘 2000 kcal」 三個數值區塊，並包含一個橫向視覺進度條。 2. 下方雙欄配置（桌面版 md:grid-cols-3）： - 左側欄（佔 2 欄）：美食搜尋區。包含一個 input 欄位與搜尋按鈕，下方預留動態渲染搜尋結果的卡片容器。 - 右側欄（佔 1 欄）：今日飲食日記清單。顯示已加入的 食物與熱量，底部顯示當前加總。 視覺風格：乾淨的健康科技風格（背景 slate-50， 主要按鈕與進度條使用 emerald-500 綠色系）。 輸出限制：請先提供完整的 HTML 與 CSS 佈局結構， JavaScript 邏輯先留白，我們稍後處理。
Prompt 2 — 生成 JavaScript 互動邏輯
基於剛才生成的 HTML 結構，請幫我撰寫原生 JavaScript： 1. 建立包含 5 筆食物的靜態 Mock Data 陣列，欄位包含 id, name, calories, tags （例如：雞胸肉沙拉 350卡、牛肉麵 650卡）。 2. 搜尋邏輯：使用者打字並點擊搜尋時，過濾符合名稱 的食物，並動態渲染到左側搜尋結果區。 3. 新增功能：點擊「加入紀錄」時，將食物 push 進 todayLog 陣列。 4. 更新邏輯：每次 todayLog 變更時，重新計算總熱量、 更新頂部儀表板數值與進度條，並動態更新右側飲食 日記清單（每個項目需包含「刪除」按鈕）。 注意：請使用資料驅動架構，定義全域 state 物件， 所有操作只修改 state，再由單一 render() 函式統一 刷新 UI。禁止在 state 以外直接操作 DOM 樣式。
⚠️

注意事項：這三個坑請小心

// expand for instructor notes
風險
AI 往往生成高度耦合的程式碼
AI 在生成原生 JS 時，常把 DOM 操作直接寫在 click event 裡，同時又修改計算數值。功能一多，程式碼就難以維護，debug 成本極高。
解法
要求 AI 採用「資料驅動」架構重構
定義全域狀態：state = { searchResults: [], todayLog: [] }
所有操作只修改 state，再由一個獨立的 render() 函式統一刷新 UI。
這能幫工程師建立良好的元件開發思維，也是 React / Vue 的核心概念。
風險
字串拼接 Tailwind 類名會讓樣式直接消失
如果 AI 寫出 class="bg-${color}-500" 這種動態拼接，Tailwind 的 CDN 掃描器無法識別完整類名，該樣式會完全失效——不報錯、靜默消失，很難發現。
解法
在 Prompt 中明確限定 AI 的寫法
遇到動態樣式時，要求 AI：
· 直接操作 element.style.width 或 element.style.backgroundColor
· 或使用完整類名進行條件切換（如 bg-red-500 vs bg-emerald-500）
· 禁止使用字串模板拼接類名。
風險
攝取超標時，進度條爆框、剩餘卡路里顯示負數
當已攝取熱量超過目標（如吃了 2500 kcal，目標 2000 kcal）時，百分比會變成 125%。
若直接將數值套入 CSS，進度條會爆框變形；剩餘卡路里也會顯示 -500 kcal。
解法
加上防禦性程式碼 + 超標警示色
進度條寬度限制：Math.min(percent, 100)
剩餘卡路里顯示：Math.max(remaining, 0) 或改顯示「已超標 XXX kcal」
超標時進度條自動變紅色（bg-red-500），給使用者明確的視覺警示。
這能訓練工程師對 AI 產出進行邊界條件審查的能力。
