## 核心觀念：傳統程式 vs 生成式 AI

要跟工程師解釋生成式 AI，最快的方法就是對比他們每天在寫的 Code：

### 1. 傳統程式（Traditional Programming）：規則驅動

在過去，我們寫程式是 **「輸入資料 + 寫死規則（If/Else、演算法）= 輸出結果」**。

* **邏輯：** 工程師扮演上帝。你必須把所有的邏輯（Logic）、邊界條件（Edge Cases）全部用程式碼定義好。如果漏寫了一個 if，系統就會噴 Bug。

### 2. 生成式 AI（Generative AI）：機率與預測驅動

生成式 AI 顛覆了這個過程。它是 **「餵入海量資料 + 模型訓練 = 機器自己找出潛在規則」**。當我們給它一個輸入（Prompt）時，它是基於這些規則，去「憑空創造（Generate）」出世界上原本不存在的新內容。

* **邏輯：** 工程師不再寫死規則，而是給予「提示與上下文」。AI 本質上是一個「超級文字接龍機」**或**「像素預測器」，它會根據你給的前文，去計算下一個字、下一個像素「最有可能出現的是什麼」，進而組合出一段程式碼、一張圖或一篇文章。

---

> 簡而言之：傳統 AI 是拿來做**分類與判斷**的（例如：判斷這封郵件是不是垃圾信、這張照片是不是貓）；而生成式 AI 是拿來**創造新東西**的（例如：幫我寫一封感謝信、畫一隻在太空的貓）。

## 工程師最需要知道的生成式 AI 三大核心底層

在課程的開頭，您可以幫工程師科普這三個核心名詞，他們聽完一定會心領神會：

### 1. 什麼是 LLM（大型語言模型）？

它是生成式 AI 的大腦（如 GPT-4, Claude 3.5）。你可以把它想像成一個「讀完了整個網際網路、壓縮過的超大型資料庫」。

* **對工程師的隱喻：** 它就像一個熟讀了 Stack Overflow、GitHub 所有開源專案、以及無數技術文件的「超級實習生」。它雖然什麼都看過，但他沒有獨立思考能力，他只是在「模仿」人類寫出正確答案機率最高的文字。

### 2. 什麼是 Token（標記）？

這是生成式 AI 計算成本與理解文字的最小單位。AI 其實看不懂中文或英文，它必須把文字切成一塊一塊的 Token，再轉成電腦懂的數字（向量）。

* **對工程師的隱喻：** 這就是為什麼 AI 計費是用 Token 計算，而不是用字數。而且這也解釋了為什麼 AI 有「Context Window（上下文視窗）」的限制——因為它的記憶體快取（Cache）是有限的，超過一定數量的 Token 它就會開始忘記前面的對話。

### 3. 什麼是「提示詞工程（Prompt Engineering）」？

對工程師來說，Prompt 其實就是一種「新型態的程式語言」。

* **對工程師的隱喻：** 過去我們用 Python 或 JavaScript 對機器下指令；現在我們用「人類的自然語言」對 AI 下指令。Prompt Engineering 的本質，就是透過精確的變數設定（Role, Context, Constraints），來**收斂 AI 的機率分布**，讓它不要瞎掰（幻覺），乖乖輸出我們要的正確結果。

## 詳細內容


### 什麼是 TOKEN


在大型語言模型中，**Token 是模型處理與運算文字的最基本單位**。當我們向模型輸入一段句子時，這段文字會先經過一個稱為 Tokenization（分詞）的過程，被切分成一個個的 Token。

關於 Token 的特性與在模型中的運作方式，包含以下幾個重要概念：

*   **與字詞的對應關係不固定：** Token 並不完全等同於人類認知中的一個「字」或一個「詞」。以中文為例，很多時候好幾個中文字合起來才會構成一個 Token，甚至有時候單一箇中文字反而會被切分成好幾塊 Token。以英文而言，字母的大小寫或前面是否帶有空格，都會被模型視為完全不同的 Token，例如「Apple」、「APPLE」以及帶有前綴空格的「 Apple」都是各自獨立的 Token。此外，標點符號如驚嘆號、雙引號等，本身也是獨立的 Token。
*   **對應唯一的數字編號（ID）：** 切分出來的每一個 Token 都會對應到一個專屬的整數編號（ID）。例如在 Llama 模型內部，編號 0 的 Token 代表的就是驚嘆號。
*   **轉換為高維度向量（Token Embedding）：** 語言模型無法直接運算文字或單純的 ID 數字，因此會透過一個稱為 Embedding Table 的龐大矩陣進行「查表」，**將每一個 Token 的 ID 轉換成一排數字，也就是一個高維度的向量（Vector）**。
*   **蘊含語意相似性：** 這些 Token 轉換成的向量並非隨機給予，**意義相近的 Token 會獲得在空間中距離相近的數學向量**。舉例來說，當模型看到「Apple」這個 Token 並將其轉換為向量後，如果去計算相似度，會發現它跟「MacBook」、「iPhone」或是中文「蘋果」的 Token 向量非常接近。
*   **語言模型的最終目標：** 語言模型運作的核心任務，就是讀取前面已經輸入的一連串 Token，經過層層的神經網路計算後，**去預測接下來的句子應該接上字典裡哪一個 Token 的機率最高**。


#### 可以用平易的觀念或是範例 解釋給我知道什麼是 Token 嗎？
我們可以把 Token 想像成是語言模型在閱讀和寫作時使用的**「文字積木」**。模型其實看不懂我們人類的完整句子，它必須先把句子拆解成一塊一塊的積木（Token），才能開始進行數學運算。

以下用幾個平易近人的範例來幫你理解 Token 的運作方式：

*   **積木是怎麼切的（不一定等於一個字）：** 
    對模型來說，積木的切法跟我們人類直覺有時不太一樣。以中文來說，雖然為了方便理解，我們常假設一個中文字就是一個 Token，但實際上，有時候好幾個中文字合起來才是一塊 Token，也有時候一個中文字會被切碎成好幾塊 Token。
    以英文來說，大小寫或是有沒有空格，都會被視為形狀完全不同的積木。例如首字母大寫的「Apple」、全部大寫的「APPLE」，以及前面多帶一個空白鍵的「 Apple」，對模型來說都是三塊各自獨立的 Token。就連標點符號，例如「驚嘆號（！）」本身也是一塊獨立的 Token。

*   **積木都有專屬編號（Token ID）：** 
    因為電腦只懂數字，所以切出來的每一塊文字積木，都會被貼上一個專屬的數字標籤（ID）。例如在知名的 Llama 模型中，編號 0 號的天字第一號 Token，其實就是「驚嘆號」。

*   **積木會互相連結（蘊含相關意義）：** 
    這些積木並不是死板板的符號，模型會把它們轉換成帶有意義的空間位置。舉例來說，當模型拿到「Apple」這塊積木時，它會發現這塊積木跟代表水果的「banana（香蕉）」靠得很近。但同時，模型也知道蘋果公司的存在，所以「Apple」這塊積木跟「MacBook」、「iPhone」、中文的「蘋果」，甚至是蘋果總部所在地「Cupertino」都非常接近。
    中文也是如此，假如你給模型一個代表姓氏「王」的 Token，模型算出來的結果會顯示它知道「王」對應到英文的「King」，同時也跟另一個中文姓氏「黃」的意思很接近。

*   **語言模型的工作就是「猜下一塊積木」：** 
    你可以把語言模型想像成一個超強的「文字接龍」機器。當你輸入一段話，模型會把這些積木綜合起來看。例如當你輸入「兩顆青蘋」，模型會考量前面這幾塊積木（兩、顆、青、蘋）的意思，並推算接下來接上「果」這塊積木的機率最高。再舉個例子，如果輸入「天氣」，模型在內部經過層層思考後，可能會預測下一個積木應該接「預」，再下一個積木接「報」，最後產出「天氣預報」這個詞。

簡單來說，**Token 就是語言模型用來拆解、理解和預測人類語言的最基礎文字積木**。


### LLM（大型語言模型）是什麼？它是怎麼運作的？

大型語言模型（LLM）本質上就是一個複雜的數學函式，**它的核心任務是接收一段未完成的句子，然後輸出字典裡每一個 Token 接在後面的機率分佈，藉此預測「下一個 Token 應該是什麼」**。

延續我們前面提到的「文字積木」概念，從你輸入一段話到模型產生回答，LLM 內部的運作可以拆解成以下幾個主要步驟：

**1. 將文字切分為 Token (Tokenization)**
首先，輸入的句子會先被切分成一個個的 Token，並各自對應到一個專屬的數字編號（ID）。

**2. 查表轉換為高維度向量 (Token Embedding)**
因為電腦無法直接處理這些數字 ID，模型內部會有一個龐大的「嵌入矩陣（Embedding Table）」。每個 Token 的 ID 都會透過這張表，被轉換成一排數字，也就是一個高維度的向​​量。在這個數學空間中，**意義相近的 Token 所對應的向​​量距離會非常接近**（例如「Apple」會跟「banana」或「iPhone」很接近）。

**3. 層層網路推敲上下文 (Layers 運作)**
轉換好的向​​量會進入模型內部的「層（Layers）」，這也就是所謂的深度學習（Deep Learning）或神經網路（Neural Network）的核心所在。大型模型通常有多個 Layer（例如 Llama 3B 有 28 層）。每一層主要會運用兩種機制來反覆處理這些向量：
*   **自注意力機制 (Self-Attention)：** 這是模型能夠「理解上下文」的關鍵。模型會去計算目前考量的 Token 與句子中前面其他 Token 之間的關聯程度（影響力）。例如輸入「兩顆青蘋果」，注意力機制會發現「青」和「兩」都會對「果」的意義產生不同面向的影響，並將這些資訊揉合在一起。在此過程中，模型也會加入「位置嵌入（Positional Embedding）」來分辨字詞的先後順序跟距離。
*   **前饋神經網路 (Feed Forward Network)：** 過完注意力機制後，向​​量會再經過一連串的矩陣乘法與活化函數（Activation Function）處理，做進一步的轉換。

經過這層層處理後，原本只代表單一字義的 Token 向​​量，就會變成**融合了整個句子上下文語意的向​​量（Contextualized Embedding）**。

**4. 計算最終得分 (LM Head)**
當向​​量跑完所有的 Layer 後，模型會拿出最後一層輸出的向​​量，將其乘上一個被稱為「LM Head」的矩陣（在許多語言模型中，這個矩陣其實跟一開始的 Embedding Table 是同一個）。這個過程等於是拿最終考慮好上下文的向量，去跟字典裡所有的 Token 向​​量計算「相似度」來打分數。**相似度越高的 Token，代表它越適合接在未完成的句子後面**。

**5. 轉換為機率並「擲骰子」 (Softmax)**
LM Head 算出來的分數（稱為 Logits）可能會有正有負，還不能當作真實的機率來用。因此，最後會經過一個名為 **Softmax** 的數學操作，將這些分數轉換成介於 0 到 1 之間、且總和為 1 的機率分佈。在此階段，系統也可以透過調整「溫度（Temperature）」參數，來決定機率分佈要越平均（讓模型偏向天馬行空的創意模式），還是越集中在少數幾個高分的字詞上（偏向保守精準）。

總結來說，LLM 就是將文字拆成積木、轉為數字向量後，在神經網路中透過「注意力機制」層層疊加對上下文的理解，最後算出每一塊積木接在後面的機率，以極度複雜的數學運算來玩「文字接龍」。

#### 可以用簡單的概念解釋 或是範例來舉例嗎

延續前面「文字積木」的概念，我們可以把大型語言模型（LLM）的運作想像成一座**「超大型的文字加工廠」**，而這座工廠唯一的工作目標，就是根據你給的未完成句子來玩「文字接龍」，預測下一個積木（Token）應該是什麼。

我們可以透過以下幾個平易近人的比喻與範例，來理解它內部的運作流程：

**1. 數十層的神經網路 ＝「流水線上的加工站」**
語言模型內部由非常多個「層（Layers）」組成，你可以把這些層想像成是**工廠流水線上的一個個站點**。每一塊文字積木進入工廠後，會一站一站地往下傳遞，每一站都會做一點處理，合起來就能完成非常複雜的語意理解。

**2. 注意力機制（Attention） ＝「員工左顧右盼，綜合判斷意思」**
在流水線的站點裡，有一個關鍵機制叫注意力機制。積木的意思往往不能單獨看，必須看上下文，所以負責處理的員工會去觀察「哪些其他的積木會影響當前積木的意思」：
*   **範例（觀察形容詞）：** 假設輸入的句子是「兩顆青蘋果」。如果只看最後一個「果」字，模型其實不知道它指的是西瓜、鳳梨還是蘋果。這時，站點裡的員工（注意力機制）會發現前面的「青」和「蘋」會深刻影響「果」的涵義，把這些資訊揉合在一起後，模型就知道這不是普通的果，而是「青色的蘋果」。
*   **範例（多個面向的觀察）：** 除了看顏色和種類，工廠裡還會有另一批專責的員工（也就是多頭注意力機制 Multi-head Attention），他們專門負責看「數量」。這批員工就會把注意力放在最前面的「兩」這個積木上，讓模型知道這是一個複數的詞彙。

**3. 考慮上下文的向量 ＝「同一個字，會隨情境變身」**
因為有了這條會互相參考的流水線，模型就能夠精準區分「長得一模一樣，但意思不同」的字。
*   **範例（區分 Apple）：** 假設有兩個句子，一句是「早餐吃了一顆 Apple（食物）」，另一句是「製造 iPad 和 AirPods 的 Apple（蘋果電腦）」。一開始剛進工廠時，這兩個 Apple 的積木長得一模一樣。但隨著它們在流水線上一層層往下走，經過員工對上下文的解讀，這兩個 Apple 在模型內部轉換成的模樣就會越來越不相像，最終模型就能完全明白它們是截然不同的兩個東西。

**4. 逐層解讀 ＝「偷窺模型的內心思考過程」**
這條流水線不只在加工，科學家甚至能透過技術，看到模型在每一層站點裡「內心的思考過程」。
*   **範例（中英轉換）：** 當我們輸入「天氣」兩個字，想讓模型接龍時，科學家發現模型在走到流水線中途的站點時，心裡其實是先浮現出英文的「weather」（天氣）與「forecast」（預報），直到經過最後一站，才決定配合使用者的語言，轉換成中文的「預」跟「報」，最終吐出「天氣預報」。
*   **範例（翻譯的內心戲）：** 當要求模型把法文的「fleur」（花）翻譯成中文時，模型在最開始幾層會吐出亂七八糟的字，但在中間幾層它其實會先想出英文的「flower」，最後才轉換並輸出中文的「花」。

**5. 轉換為機率 (Softmax) ＝「最後的擲骰子遊戲」**
當積木走完全部的流水線站點，來到最後一關時，模型會對字典裡幾萬個積木進行打分，並轉換成機率。最後，系統會根據這些機率來**「擲骰子」**，決定下一個要派出的積木是哪一個。
*   我們在使用語言模型時常聽到的「溫度（Temperature）」參數，就是在控制這顆骰子。如果把溫度調高，骰子的機率分佈會比較平均，模型就有機會擲出比較罕見、奇特的字，這就是很多模型標榜的**「創意模式」**；反之則會給出最保守、最常見的答案。
