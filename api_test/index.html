把 Mock Data 換成真實資料——呼叫營養成分 API、解析 JSON、處理 loading 與錯誤，完成完整的資料串接鏈。

⚡

Prompt 範例

// copy & paste into Claude
Prompt 1 — 生成 API 呼叫程式碼
我要用原生 JavaScript 的 fetch 呼叫以下 API： API 端點： https://world.openfoodfacts.org/cgi/search.pl 請求方式：GET 必要參數（Query String）： - search_terms: 使用者輸入的食物名稱 - json: 1（回傳 JSON 格式） - page_size: 5（最多 5 筆結果） 回傳 JSON 的關鍵結構如下： { "products": [ { "product_name": "Chicken Breast", "nutriments": { "energy-kcal_100g": 165, "proteins_100g": 31, "carbohydrates_100g": 0, "fat_100g": 3.6 } } ] } 請幫我： 1. 撰寫一個 async function searchFood(query)， 呼叫此 API 並回傳解析後的食物清單陣列。 2. 每筆資料只保留：name、kcal、protein、carbs、fat （注意某些欄位可能是 undefined，請給預設值 0）。 3. 加上 try/catch 錯誤處理，錯誤時 throw 讓呼叫方處理。
Prompt 2 — 渲染結果 + 完整狀態處理
基於剛才的 searchFood() 函式，請幫我撰寫完整的 搜尋互動功能： 1. handleSearch() 函式： - 取得輸入框的 query 字串 - 在呼叫 API 前顯示 loading 狀態 - 呼叫 searchFood(query) - 成功：呼叫 renderResults(foods) - 失敗：顯示錯誤訊息「API 呼叫失敗，請稍後再試」 2. renderResults(foods) 函式： - foods 為空時：顯示「查無結果，請嘗試其他關鍵字」 - foods 有資料時：渲染成卡片列表，每張卡片顯示： 食物名稱、熱量(kcal/100g)、蛋白質(g)、碳水(g)、脂肪(g) - 每張卡片有一個「加入紀錄」按鈕 3. 防止 Race Condition： 若使用者快速點擊搜尋，請使用 AbortController 取消 上一個未完成的請求，避免舊結果覆蓋新結果。 請使用 async/await 語法，不要用 .then() 鏈。
⚠️

注意事項：API 串接的三大陷阱

// expand for details
風險
瀏覽器的 Same-Origin Policy 會阻擋跨域請求
若第三方 API 沒有設定 Access-Control-Allow-Origin: *，瀏覽器就會拒絕請求並拋出 CORS 錯誤。
這個錯誤不是你的程式碼壞了，是瀏覽器的安全機制阻擋了你。
解法
選支援 CORS 的 API，或透過自己的後端代理
· 短期：選擇明確支援 CORS 的 API（如 Open Food Facts、JSONPlaceholder）
· 長期：在自己的 Node.js 後端建立代理路由，前端只呼叫自己的後端，由後端轉發 API 請求
· CORS 是後端設定的，前端無法自行解決
風險
API Key 寫在 JS 裡，任何人開 DevTools 都能看到
把 API Key 直接寫在前端 JS 裡（如 const apiKey = 'sk-abc123...'），任何人按下 F12 就能取得你的 Key，並用它無限呼叫 API 消耗你的配額或費用。
解法
API Key 放後端，前端只呼叫自己的後端
· API Key 存在後端的環境變數（.env）中
· 前端只呼叫 /api/search?q=apple（自己的後端）
· 後端帶著 Key 去呼叫真實的第三方 API，再回傳結果
· 這樣前端永遠不會看到 Key
風險
使用者快速搜尋時，先發出的請求可能更晚回應
使用者輸入「chicken」，請求 A 發出；馬上改成「beef」，請求 B 發出。
若請求 A 因為某些原因比請求 B 晚回來，畫面就會顯示「chicken」的結果，但搜尋框顯示的是「beef」。
解法
用 AbortController 取消舊請求，或用 debounce 延遲觸發
· AbortController：每次發新請求前，先 abort() 上一個未完成的請求
· debounce：使用者停止輸入 300ms 後才觸發搜尋，減少請求數量
· 兩者可以同時使用以獲得最佳效果
